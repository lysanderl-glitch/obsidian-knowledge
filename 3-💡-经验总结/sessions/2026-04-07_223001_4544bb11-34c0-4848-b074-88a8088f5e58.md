| 层次 | 含义 | Claude Code对应 |
|------|------|----------------|
| **做正确的事** | 先确定目标/方向是对的 | 探索→计划，先理解问题再动手 |
| **正确的做事** | 用正确的方法/流程 | Plan Mode、验证方式、模块化 |

```
官方：Explore first, then plan, then code
```

**助手**: 对，这正是 Harness Engineering 的核心思想。

**Harness Engineering 本质上就是"元原则工程"：**

| 层次 | Harness 对应 |
|------|-------------|
| **元原则** | 做正确的事、正确的做事、把事情做正确 |
| **前馈控制（Harness）** | 把正确做事的方法固化到系统里，让 AI 第一次就做对 |
| **反馈控制** | 错误发生后的修正机制（知识库 + AI 分析） |

**你正在做的：**
- 调试 workflow 时积累"做正确的事"的经验（先验证 Slack API 是否通）
- 把这些经验固化到知识库形成"元原则"
- 下次遇到同类问题，AI 直接应用原则而非重复试错

**本质是：让 AI 从错误中学习，但学的是"如何不犯同类错"的元知识，而不是具体错误的修复方法。**

这也是 Mitchell Hashimoto 说的：每犯一个错误，花时间设计一个解决方案确保永远不再犯。

要把这个认知也补充到 Harness 知识库里吗？
