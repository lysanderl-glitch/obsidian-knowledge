## 2026-04-07 案例1：Harness Self-Healing 系统调试

**问题：** Python 分析服务器 (harness-analyze-server.py) 调用 MiniMax Claude API 时返回空 analysis

**错误日志：**
```
TimeoutError: The read operation timed out
```

**根因分析：**
1. MiniMax-M2 模型返回的 content 数组结构：第一个元素是 `type: "thinking"`（长思考内容），第二个才是 `type: "text"`（实际回答）
2. 原代码只取第一个 type="text" 的元素，但 thinking 在前面，导致取到空值
3. 另外 max_tokens=300 不足以返回完整 JSON（包含 suggested_fix 等长字段）

**解决方案：**
```python
# 1. 禁用 thinking 模式
api_payload = {
    "model": "claude-3-5-sonnet-20241022",
    "max_tokens": 500,
    "thinking": {"type": "disabled"},  # 关键修复
    "messages": [{"role": "user", "content": prompt}]
}

# 2. 确保端口可复用
socketserver.TCPServer.allow_reuse_address = True
```

- Category: permission (正确分类)
- Analysis: 完整的根因分析文本
- Suggested Fix: 具体修复步骤
