# 项目管理自动化方案：项目启动 → Asana 工序创建 → Slack 提醒

创建时间：2026-04-03

## Context

用户需要一套标准化的项目执行自动化方案。场景：项目启动时，根据工序表在 Asana 创建工序任务，设置负责人和依赖关系；前置任务完成后自动 Slack 提醒下一步负责人。

**输入输出：**
- 触发：Asana 中「项目启动」任务被创建
- 输入：Excel/CSV 工序表（知识库存储）
- 输出：Asana 工序任务（含依赖）+ Slack 项目频道通知

---

## 方案架构

```
┌──────────────┐     ┌─────────────────┐     ┌────────────────┐
│  知识库 Git   │────▶│  工作流 A        │────▶│     Asana      │
│  工序表文件   │     │  项目启动        │     │  创建工序任务   │
└──────────────┘     └────────┬────────┘     └────────────────┘
                              │ Asana Webhook
                              ▼
                       ┌─────────────────┐     ┌────────────────┐
                       │  工作流 B        │────▶│     Slack      │
                       │  工序完成提醒    │     │  项目专用频道   │
                       └─────────────────┘     └────────────────┘
```

## 工序表设计

**存储位置：** `/home/ubuntu/knowledge-base/process-tables/process-{项目标识}.csv`

**数据结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 序号 | int | 唯一标识，用于依赖引用 |
| 工序名称 | string | 任务名称 |
| 负责人邮箱 | string | Asana 用户邮箱，用于查 User ID |
| 持续时间(天) | int | 估算工期，计算 due_on |
| 前置工序序号 | string | 逗号分隔，如 "1,2" |
| Slack频道 | string | 项目专用频道，如 #proj-alpha |

**示例：**
```
序号,工序名称,负责人邮箱,持续时间(天),前置工序序号,Slack频道

3,前端开发,wissame@janusd.com,5,2,#proj-alpha
```

## 工作流设计

### 工作流 A: 项目启动 → 创建工序任务

**触发：** Asana Webhook（监听「项目管理」项目中的任务创建）

**节点链：**
```
Asana Trigger → 解析任务信息 → 读取工序表 → 解析CSV → 批量查询用户ID → 循环创建任务 → 批量设置依赖 → 创建Slack频道
```

**关键配置：**
- `executeOnce: false` 在循环创建任务节点
- 依赖设置通过 Code 节点构建 + HTTP Request 调用 `POST /tasks/{gid}/addDependencies`
- Slack 频道创建使用 Slack Node `channel.create`

---

### 工作流 B: 工序完成 → Slack 提醒

**触发：** Asana Webhook（监听所有任务 completed 字段变化）

**节点链：**
```
Asana Trigger → IF(完成且是工序任务) → 获取任务详情 → 查询下游依赖任务 → 获取负责人 → 发送Slack通知
```

**关键配置：**
- IF 节点过滤：`completed == true && 是工序任务（非项目启动本身）`
- Slack 消息使用 Block Kit 格式，包含：完成任务名称、下一步工序、负责人、链接

---

## Slack 频道策略

**命名：** `#proj-{项目标识}`
**创建：** 工作流 A 自动创建（private channel）
**映射存储：** 在 Asana「项目启动」任务的 notes 字段记录 `SLACK_CHANNEL: #proj-alpha`

---

## 实施步骤

1. **准备工序表模板**
   - 在知识库创建 `process-tables/` 目录
   - 提供 CSV 模板文件

2. **创建工作流 A**
   - Asana Trigger 监听任务创建
   - HTTP Request 读取工序表
   - Spreadsheet 节点解析 CSV
   - Split in Batches + Asana 批量创建任务
   - Code 节点构建依赖映射 + HTTP Request 设置依赖
   - Slack Channel Create 创建项目频道

3. **创建工作流 B**
   - Asana Trigger 监听任务更新
   - IF 过滤 completed 事件
   - Asana Get Task 获取详情
   - Asana Search 查询下游任务
   - Slack Post Message 发送通知

4. **测试验证**
   - 手动触发工作流 A 测试工序创建
   - 完成单个工序测试工作流 B 通知
