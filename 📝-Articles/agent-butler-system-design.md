> 状态：初稿，待评审

## 一、系统愿景

构建一个 **Agent 协作操作系统**，实现：
- **管家 Agent**：担任调度协调角色，负责任务分配、决策判断、进度监控
- **专家 Agent 集群**：各领域专家（微信运营、博客发布、Slack 通知等）
- **自动决策**：管家可判断的自动执行，需人决策的上报
- **知识沉淀**：能力库、决策记录、经验积累

## 二、核心架构

### 2.1 Agent 定义规范（参考 CrewAI）

每个 Agent 包含：

| 属性 | 说明 | 示例 |
|------|------|------|
| **role** | 角色/专业领域 | "微信文章运营专家" |
| **goal** | 具体目标 | "生成符合微信公众号规范的精美文章" |
| **backstory** | 背景/能力描述 | "5年新媒体运营经验，精通微信公众号生态" |
| **tools** | 工具能力 | [fetch_article, convert_html, create_draft] |
| **memory** | 记忆/上下文 | 任务历史、用户偏好 |
| **llm** | 使用的模型 | claude-sonnet |

## 三、决策系统设计

### 3.1 决策类型

| 决策类型 | 描述 | 处理方式 |
|----------|------|----------|
| **L1 自动** | 例行操作、已验证流程 | 管家直接执行 |
| **L2 审批** | 需要确认的方案 | 管家判断，通过则执行 |
| **L3 上报** | 复杂/风险/未定义 | 通知用户决策 |

### 3.2 决策规则示例

```yaml
# 决策规则配置
rules:
  - condition: "任务类型 == '发布微信文章' AND 方案已验证"
    level: L1
    action: 自动执行
  
  - condition: "任务类型 == '发布微信文章' AND 首次执行"
    level: L2
    action: 管家审批方案后执行
  
  - condition: "涉及外部支付 OR 不可逆操作"
    level: L3
    action: 上报用户
  
  - condition: "超出管家能力范围"
    level: L3
    action: 上报用户
```

### 3.3 权限自动授权机制

**问题**：每次访问新服务都要询问权限

**解决方案**：建立权限矩阵

```yaml
# 权限配置
permissions:
  # 信任级别：system > user_confirmed > untrusted
  trust_levels:
    - system: "已验证的系统操作，自动授权"
    - user_confirmed: "用户明确授权的操作，自动授权"  
    - untrusted: "新服务/风险操作，需确认"

  # 具体权限
  operations:
    - op: "wechat.create_draft"
      trust_level: user_confirmed
      description: "创建微信草稿，自动通过"
    
    - op: "wechat.publish"
      trust_level: user_confirmed  
      description: "发布微信文章，需 L2 审批"
    
    - op: "slack.send_message"
      trust_level: system
      description: "发送 Slack 消息，系统操作"
    
    - op: "external.unknown"
      trust_level: untrusted
      description: "未知外部服务，需用户明确授权"
```
